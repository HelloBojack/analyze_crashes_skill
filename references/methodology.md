# 股票暴跌分析方法论

## 分析框架

### 1. 数据获取

支持两种数据源，优先使用 westock-data，fallback 使用 yfinance：

**方式一：westock-data（推荐 WorkBuddy 用户）**
- 自动探测 westock-data 脚本路径（环境变量 `WESTOCK_DATA_SCRIPT` 或默认安装路径）
- 自动探测 node 路径（环境变量 `NODE_PATH` 或系统 PATH）

**方式二：yfinance（无需 WorkBuddy）**
- `pip install yfinance`
- 自动从 Yahoo Finance 获取日线数据

支持任意美股代码（如 AAPL, TSLA, NVDA）和对比指数（如 QQQ, SPY）

### 2. 暴跌定义
- **阈值**：日跌幅 > 指定百分比（默认 2%）
- 基于前一交易日收盘价计算当日收益率

### 3. 原因分类（三层）

根据对比指数（如 QQQ）当天的表现判断暴跌原因：

| 分类 | 条件 | 含义 |
|------|------|------|
| **自身导致** | 对比指数 ≥ -0.3% | 大盘平稳或上涨，股票独自下跌，通常是自身利空 |
| **整体下挫** | 对比指数 < -1.5% | 大盘同步暴跌，市场整体恐慌 |
| **整体偏空+自身弱** | -1.5% ≤ 对比指数 < -0.3% | 大盘偏弱，但股票跌幅更大，双重因素 |

### 4. 修复定义
- **修复**：收盘价重新超过暴跌前一日的收盘价
- **修复天数**：从暴跌日（含）到修复日（含）的交易日数
- 未修复：在数据范围内仍未回到暴跌前水平

### 5. 统计指标

#### 稳健统计量（推荐）
- **中位数**：中间值，不受极值影响
- **25%分位数（较快修复）**：25%的事件在这个天数内修复
- **75%分位数（较慢修复）**：75%的事件在这个天数内修复
- **90%分位数**：90%的事件在这个天数内修复

#### 参考统计量
- **平均值**：仅供参考，易被极值拉高
- **最快/最慢**：修复天数的范围

### 6. 修复概率

计算不同时间窗口内的修复概率：
- 2天内 / 3天内 / 5天内 / 7天内 / 10天内 / 30天内 / 60天内

概率 = 该窗口内修复的次数 / 总修复次数（或总次数）

### 7. 跌幅分层分析

将暴跌按跌幅分为四层：
- 2-3% / 3-4% / 4-5% / >5%

每层独立计算：
- 修复概率
- 修复天数分位数
- 次数分布

### 8. 投资关键指标

- **V型修复率**：5天内修复的比例
- **暴跌后5天平均收益**：均值回归策略参考
- **5天收红概率**：抄底后赚钱的概率
- **修复日反弹幅度**：修复当天平均涨多少
- **自身原因 vs 市场原因修复差距**：识别该跑还是该拿

### 9. 事件数据（可选）

可以传入外部 JSON 事件数据来补充具体事件描述：

```json
{
  "2025-03-11": "彭博报道苹果推迟Siri AI升级发布",
  "2026-06-25": "苹果宣布全线产品涨价"
}
```

不传事件数据时，事件描述会根据原因分类自动生成通用描述。

### 10. 输出格式

生成交互式 HTML 报告，包含：
- 概览统计卡片
- SVG 图表（原因分布圆环图、修复天数柱状图、月度分布）
- 修复概率统计
- 按原因分类修复统计
- 按跌幅分层修复统计
- 月度分布
- 投资关键洞察
- 三个可筛选排序的表格

## 使用建议

1. **默认参数**：阈值 2%，对比指数 QQQ，适合大多数美股分析
2. **A股分析**：对比指数建议改为对应的大盘指数（如 上证指数/深证成指）
3. **事件补充**：对重要股票建议补充具体事件数据，提高分析质量
4. **阈值调整**：
   - 高波动股票（如 TSLA, NVDA）建议用 3% 或更高
   - 低波动股票（如 银行股）可以用 1.5%
